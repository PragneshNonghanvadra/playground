# Prompts

Reusable prompts that have worked well in this repo. Treat them as living documents.

## Conventions

- Each prompt is a markdown file: `kebab-case-name.md`.
- Top of the file: purpose, when to use, expected output.
- Bottom of the file: variations, pitfalls, dated revisions.

## Suggested starter prompts to add over time

- `draft-topic-note.md` — generate a first draft of a topic README from a one-line description.
- `critique-architecture.md` — adversarial review of an ADR or RFC.
- `enumerate-failure-modes.md` — produce a candidate list of failures for a given system.
- `interview-explainer.md` — produce 30-second / 2-minute / deep-dive explanations of a topic.
- `eval-rubric.md` — propose a rubric for an AI feature.

> Stub. Add prompts here as patterns emerge.
