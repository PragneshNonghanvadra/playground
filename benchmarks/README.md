# benchmarks/

Performance, cost, and correctness benchmarks across stacks. Benchmarks are **for learning**, not for marketing — capture honest numbers, including the bad ones.

## Subfolders

- [`frontend/`](frontend/)
- [`api/`](api/)
- [`db/`](db/)
- [`ai/`](ai/)
- [`infra/`](infra/)

## Conventions

- Each benchmark has a `README.md` describing workload, target metric, hardware, and methodology.
- Results are committed alongside the script that produced them.
- Always include error bars or runs-of-N. Single numbers lie.
