# Learning Principles

These are the rules I use to study engineering deeply. Re-read them quarterly.

## 1. Learn concepts before tools

Understand why a technology exists before learning its API. Tools change every few years; the underlying problems they solve do not.

## 2. Start with first principles

Explain the problem without naming the tool. If I can't describe the problem in plain language, I don't understand it.

## 3. Write before building

Document the mental model, tradeoffs, and failure modes before implementation. Writing exposes shallow understanding faster than coding does.

## 4. Build tiny experiments

Small experiments are better than large unfinished projects. A working 100-line lab beats a half-built monolith every time.

## 5. Measure something

Every serious lab should measure latency, throughput, correctness, cost, memory, reliability, or UX. "I think it's fast" is not learning.

## 6. Break the system intentionally

Every non-trivial lab should include a failure experiment. If I haven't broken it, I haven't really learned how it works.

## 7. Connect concepts

Each topic should link to related topics. Knowledge compounds when it forms a graph, not a list.

## 8. Keep interview notes

If a concept is important professionally, write how to explain it in interviews — 30 seconds, 2 minutes, deep dive. Forces clarity.

## 9. Review AI-generated work critically

AI can accelerate implementation, but it should not own architectural judgment. Critique, override, log. The judgment is the point.

## 10. Prefer durable knowledge

Prioritize concepts that remain useful beyond current frameworks. HTTP, transactions, queues, indexing, caching, distributed consensus — these outlast any framework.

## 11. Re-read your own notes

If a note is worth writing, it's worth coming back to. Schedule re-reads. Revise. Confidence levels should evolve.

## 12. Honesty over polish

Capture uncertainty. "I don't know yet" and "I think but haven't verified" are valuable signals. Polished-but-wrong is worse than rough-but-honest.
