# C4 Models

C4-model views of major systems: context, container, component, code.

## Conventions

- One folder per system: `c4/<system-name>/`.
- Inside, four files: `1-context.md`, `2-container.md`, `3-component.md`, `4-code.md` (the last is optional).
- Each file includes a Mermaid or ASCII diagram and a short description.

## Why C4

C4 makes architecture diagrams **navigable** rather than overwhelming. A single page rarely captures a real system; a layered set does.
