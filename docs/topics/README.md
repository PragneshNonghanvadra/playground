# Topics

One folder per engineering area. Each topic folder contains a standard set of notes — see [`templates/topic-template.md`](../../templates/topic-template.md) for the structure.

## Categories

| Category                                           | Status         | What it covers                                                    |
| -------------------------------------------------- | -------------- | ----------------------------------------------------------------- |
| [core-engineering/](core-engineering/)             | Light scaffold | HTTP, DNS, TLS, concurrency, memory, data structures, algorithms  |
| [frontend-architecture/](frontend-architecture/)   | Light scaffold | Component models, state, routing, hydration, rendering tradeoffs  |
| [backend-systems/](backend-systems/)               | **Priority**   | APIs, auth, validation, transactions, idempotency, jobs, webhooks |
| [distributed-systems/](distributed-systems/)       | **Priority**   | Queues, retries, consistency, consensus, replication, sagas       |
| [data-systems/](data-systems/)                     | **Priority**   | SQL, indexes, query plans, migrations, replication, vector        |
| [ai-engineering/](ai-engineering/)                 | **Priority**   | LLMs, prompts, RAG, evals, tool calling, cost, safety             |
| [ai-agents-workflows/](ai-agents-workflows/)       | Light scaffold | Agent loops, durable workflows, tool sandboxing, HITL             |
| [infrastructure-devops/](infrastructure-devops/)   | Light scaffold | Docker, CI/CD, IaC, deploys, secrets, networking                  |
| [observability/](observability/)                   | **Priority**   | Logs, metrics, traces, SLOs, alerts, AI traces                    |
| [security/](security/)                             | Light scaffold | OWASP, auth security, tenant isolation, prompt injection          |
