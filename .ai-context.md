# AI Context

This repository is a long-term engineering learning lab called **Playground**.

Do not assume this is a production app.

## Before making changes

1. Read the root [`README.md`](README.md).
2. Read [`ai-agent/instructions.md`](ai-agent/instructions.md).
3. Read [`ai-agent/coding-agent-rules.md`](ai-agent/coding-agent-rules.md).
4. Identify the relevant topic under [`docs/topics/`](docs/topics/).
5. Use templates from [`templates/`](templates/).
6. Keep changes small and reviewable.
7. Document reasoning, tradeoffs, and failure modes.
8. Do **not** add heavyweight infrastructure or frameworks unless requested.
9. Prefer learning value over code volume.

## What "good" looks like in this repo

- A new or updated markdown note that captures a mental model, tradeoffs, and failure modes.
- A tiny lab that demonstrates one concept clearly, with a measurement and a failure experiment.
- An ADR that names alternatives and reversal triggers.
- An interview-notes file that distills a concept into 30-second, 2-minute, and deep-dive forms.

## What "bad" looks like in this repo

- Scaffolding a full Next.js app, microservices, Kafka, or Kubernetes "just in case".
- Adding LLM abstractions without evals, logs, and failure-mode notes.
- Generating long files of boilerplate that obscure the underlying concept.
- Suppressing or hiding uncertainty.
- Skipping the topic notes and going straight to code.

## When uncertain

Stop and ask. Or capture the uncertainty in [`docs/concept-map/open-questions.md`](docs/concept-map/open-questions.md) and proceed with the smallest reversible step.
