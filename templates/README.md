# Templates

Source-of-truth templates used across the repo. Copy a template when starting new work; don't reinvent structure.

## Templates

- [`topic-template.md`](topic-template.md) — for a new topic note
- [`lab-template.md`](lab-template.md) — for a new lab
- [`adr-template.md`](adr-template.md) — for a new ADR (Architecture Decision Record)
- [`rfc-template.md`](rfc-template.md) — for a new RFC (Request for Comments)
- [`interview-template.md`](interview-template.md) — for an interview note
- [`resource-template.md`](resource-template.md) — for a resource note (book, course, paper, etc.)
- [`tradeoff-template.md`](tradeoff-template.md) — for a tradeoff comparison
- [`failure-mode-template.md`](failure-mode-template.md) — for a failure-mode analysis
- [`production-readiness-template.md`](production-readiness-template.md) — for production-readiness review
- [`ai-agent-review-template.md`](ai-agent-review-template.md) — for logging AI-agent runs
- [`system-design-template.md`](system-design-template.md) — for system-design practice / proposals

## Quality bar

These templates define the _minimum_ shape. Add sections when needed. Never delete sections without leaving a note explaining why.

If a template starts to feel wrong, update the template — don't drift away from it silently.
